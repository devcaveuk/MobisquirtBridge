Overview

MobisquirtBridge turns an inexpensive ESP32 module into a wireless gateway between your MegaSquirt (or any MegaSquirt-compatible) ECU and TunerStudio / Mobisquirt. It exposes the ECU's serial port over Wi-Fi (TCP) and optionally over Bluetooth LE, so you can tune and datalog from a phone or laptop without a USB cable.

Wi-Fi

TCP Socket Bridge

Connects on port 9001. Works in both AP mode (module creates its own network) and STA mode (joins your existing Wi-Fi). Automatically falls back to AP if STA fails.

BLE

Bluetooth LE Bridge

Optional BLE characteristic bridge. PIN-protected pairing. One client at a time shared with the TCP bridge.

Config

Web UI

Built-in configuration page on port 80. Set SSID, password, baud rate, BLE UUIDs, and more without reflashing.

Choose a Board

Both boards cost a few dollars and are supported by the provided PlatformIO firmware. Pick based on size, available GPIO, or personal preference.

Connection Options

Choose how the bridge physically connects to your ECU's serial port:

Embedded

Option A — Direct UART

Solder directly to the ECU PCB's UART pads (TX / RX / GND / 3.3 V or 5 V). Ideal for a permanent, compact installation inside the ECU enclosure.

⚠ Requires access to ECU UART pads. May need 5 V→3.3 V level shifting on RX.

External

Option B — RS-232 / DB9

Use a MAX3232 RS-232 transceiver to connect to the ECU's existing DB9 serial port. Works with any MegaSquirt that has an external serial connector.

No ECU disassembly required. Can be housed in a small project box.

5 V UART caution: The ESP32 GPIO pins are not 5 V-tolerant. If the ECU outputs 5 V logic on its TX line (which becomes the ESP32's RX line), you must shift it down to 3.3 V. A simple resistor divider is sufficient at these low frequencies.

The ESP32's TX output (3.3 V logic HIGH) is accepted as a valid logic HIGH by 5 V UART receivers, so no level shifting is needed on the TX side.

Option B — RS-232 / DB9 Build External

Adds a MAX3232 (or MAX232) RS-232 transceiver between the ESP32 and the ECU's DB9 serial port. The ECU communicates at TTL levels internally; the DB9 port uses RS-232 voltage levels (±3 V to ±15 V). The MAX3232 handles the translation transparently.

Power & Level Shifting

Powering the ESP32

Both boards accept 5 V on the USB / 5V pin and regulate it internally to 3.3 V. You can power them from:

  • USB-C port (flashing or bench use)
  • ECU 5 V rail via the 5V pin
  • 3.3 V directly into the 3V3 pin (bypass regulator)

GPIO Voltage Limits

All ESP32 GPIO pins are 3.3 V maximum. Applying 5 V to a GPIO input will damage the chip over time.

  • Direct UART from 3.3 V ECU → no shifting needed
  • Direct UART from 5 V ECU → shift RX with resistor divider
  • RS-232 via MAX3232 → MAX3232 handles all level conversion

Resistor Divider Calculator

For a 5 V → 3.3 V divider: choose R1 = 10 kΩ and R2 = 22 kΩ. Output = 5 × 22/(10+22) = 3.44 V, within the 3.3 V GPIO spec. Both values are common E24 series resistors.

MAX3232 Power Switch Circuit

The firmware holds the MAX3232 power-control GPIO LOW during boot and drives it HIGH only after full initialization. This prevents the RS-232 driver from outputting undefined signals on the DB9 while the ESP32 is starting up.

Because the MAX3232 draws ~10 mA from its supply, a GPIO pin cannot switch it directly. A simple NPN transistor provides the necessary current switching.

Power GPIO: GPIO 9 (ESP32-S3)  /  GPIO 10 (ESP32-C3)

Circuit

  3.3 V rail ─────────────────────────── MAX3232 VCC

  MAX3232 GND ──→ Collector ┐
                            │ NPN (2N3904 / 2N2222 / BC547)
                Base ──[R]──┘
                │             R = 1 kΩ to 10 kΩ (5 kΩ preferred)
  ESP32 GPIO ───┘

                Emitter ──→ GND

When the ESP32 pulls GPIO HIGH (3.3 V), the transistor saturates, completing the ground path for the MAX3232 and powering it on. When LOW (or during boot), the transistor is off and the MAX3232 is unpowered.

Transistor Pin Guide (TO-92 package)

Pin 1 (E)
Emitter → GND
Pin 2 (B)
Base → resistor → ESP32 GPIO
Pin 3 (C)
Collector → MAX3232 GND

2N3904 flat-face order (left to right): E – B – C. 2N2222 (TO-92): E – B – C. BC547 (TO-92): C – B – E (reversed — check datasheet).

Status LED

The onboard LED gives a quick health indicator without needing a serial monitor:

LED State Meaning
Blinking (600 ms) Waiting for a client (TCP or BLE)
Solid ON Client connected — bridge is active
Off Initializing or error

ESP32-C3 SuperMini

GPIO 8, active-low, standard GPIO LED. Override with STATUS_LED_PIN in platformio.ini.

ESP32-S3 Zero

GPIO 21, NeoPixel (WS2812B) RGB LED. Firmware lights it as white/off. Override behaviour with STATUS_LED_PIN and STATUS_LED_IS_NEOPIXEL.

Flash Firmware

Prerequisites

  • PlatformIO Core or the VS Code PlatformIO extension
  • USB-C cable (data-capable, not charge-only)
  • The MobisquirtBridge source repository
  1. Connect the ESP32-C3 SuperMini to your computer via USB-C.
  2. Open a terminal in the project root directory.
  3. Run: 
    pio run -e esp32-c3-supermini --target upload
  4. Monitor serial output (optional): 
    pio device monitor -e esp32-c3-supermini
  5. Connect to the MobisquirtBridge Wi-Fi network (password: mobisquirt123) and open http://192.168.4.1 to configure.
  1. Connect the ESP32-S3 Zero to your computer via USB-C.
  2. Open a terminal in the project root directory.
  3. Run: 
    pio run -e esp32-s3-zero --target upload
  4. Monitor serial output (optional): 
    pio device monitor -e esp32-s3-zero
  5. Connect to the MobisquirtBridge Wi-Fi network (password: mobisquirt123) and open http://192.168.4.1 to configure.
Default settings after first flash:
Wi-Fi mode: AP  |  SSID: MobisquirtBridge  |  Password: mobisquirt123  |  TCP port: 9001  |  Baud: 115200  |  BLE: Off

TunerStudio / Mobisquirt Connection

  1. Join the MobisquirtBridge Wi-Fi network (or configure STA mode in the web UI).
  2. In TunerStudio, create a new project and select TCP/IP as the communications type.
  3. Enter the bridge IP address (AP mode: 192.168.4.1) and port 9001.
  4. Set the baud rate to match your ECU (default 115200).
  5. Connect — the status LED should go solid.
